# Code of Conduct

This is a personal open-source project. The community spaces involved are limited to GitHub issues, GitHub discussions (if enabled), and any direct contact with the maintainer.

## Expected behavior

When interacting via issues or direct contact:

- Be respectful. Disagreements are fine; rudeness is not.
- Be specific. Vague complaints are hard to act on.
- Be patient. Response times reflect a personal-project cadence (see `SECURITY.md` for the explicit response window on security issues; other issues have no SLA).
- Stay on topic. Issues are for bugs, documentation clarity, compatibility problems, security concerns, and adaptation notes (see `CONTRIBUTING.md` for the welcome categories).

## Unacceptable behavior

- Harassment, discrimination, or personal attacks toward the maintainer or any other commenter
- Off-topic or commercial content
- Demands or expectations of unpaid labor from the maintainer
- Repeated reopening of issues closed with rationale

## Enforcement

The maintainer will close issues, lock threads, and block accounts as needed. No formal process; personal judgment applies.

## Scope

This Code of Conduct applies to GitHub interactions related to this repository. It does not apply to forks, derivative works, or use of the MIT-licensed framework in other contexts.

## Attribution

This Code of Conduct is intentionally minimal. It does not adapt the Contributor Covenant or any other formal document because this is a personal project with a single maintainer and a no-PR posture. If you need a more formal CoC for adapting this framework in your own project, the Contributor Covenant (https://www.contributor-covenant.org/) is a good starting point.
